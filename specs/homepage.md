# Feature: Homepage

## Why

The homepage establishes authority, provides navigation entry points, and captures branded searches for "haunted places directory."

## Requirements

- [ ] Route: `/`
- [ ] Hero section with tagline and search bar
- [ ] Stats bar: total places, states covered, categories
- [ ] Featured states section with cards (GA, LA, MA, PA)
- [ ] Featured places section (4-6 most iconic locations)
- [ ] Category quick links (hotels, mansions, battlefields, etc.)
- [ ] Simple search functionality (search by name/city)
- [ ] Meta tags and Open Graph for homepage
- [ ] Mobile responsive design
- [ ] Fast load time (minimal JS)

## User Flow

1. User lands on homepage
2. User either searches, clicks a state, or clicks a featured place
3. User navigates deeper into the directory

## Not Included

- Advanced filters on homepage (V2)
- User accounts (V2)
- Newsletter signup (V2)
